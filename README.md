# 1.3.4-Semin

Тест-кейсы для формы валидации email

Блок 1. Позитивные сценарии (валидные email-адреса)
1. Ввод валидного email с латинскими буквами

Шаги:

Ввести в поле ввода user@example.com
Нажать кнопку «Проверить»
Ожидаемый результат: появляется alert с текстом «Прекрасный email-адрес»

2. Ввод валидного email с цифрами в локальной части

Шаги:

Ввести user123@domain.ru
Нажать кнопку «Проверить»
Ожидаемый результат: alert «Прекрасный email-адрес»

3. Ввод валидного email с допустимыми спецсимволами

Шаги:

Ввести user!name#mail@example.org
Нажать кнопку «Проверить»
Ожидаемый результат: alert «Прекрасный email-адрес»

4. Ввод валидного email с точкой в локальной части

Шаги:

Ввести first.last@sub.domain.com
Нажать кнопку «Проверить»
Ожидаемый результат: alert «Прекрасный email-адрес»

5. Ввод валидного email с дефисом в домене

Шаги:

Ввести user@my-domain.co.uk
Нажать кнопку «Проверить»
Ожидаемый результат: alert «Прекрасный email-адрес»

Блок 2. Негативные сценарии (невалидные email-адреса)

6. Отсутствие символа «@»

Шаги:

Ввести userexample.com
Нажать кнопку «Проверить»
Ожидаемый результат: под полем ввода (над кнопкой) появляется красная ошибка «Введен некорректный email»

7. Отсутствие домена после «@»

Шаги:

Ввести user@
Нажать кнопку «Проверить»
Ожидаемый результат: ошибка «Введен некорректный email»

8. Отсутствие локальной части до «@»

Шаги:

Ввести @example.com
Нажать кнопку «Проверить»
Ожидаемый результат: ошибка «Введен некорректный email»

9. Наличие недопустимых символов (кириллица)

Шаги:

Ввести пользователь@mail.ru
Нажать кнопку «Проверить»
Ожидаемый результат: ошибка «Введен некорректный email»

10. Наличие пробелов внутри email

Шаги:

Ввести user name@domain.com
Нажать кнопку «Проверить»
Ожидаемый результат: ошибка «Введен некорректный email»

11. Две точки подряд в домене

Шаги:

Ввести user@domain..com
Нажать кнопку «Проверить»
Ожидаемый результат: ошибка «Введен некорректный email»

12. Отсутствие точки в домене

Шаги:

Ввести user@domaincom
Нажать кнопку «Проверить»
Ожидаемый результат: ошибка «Введен некорректный email»

13. Email заканчивается точкой

Шаги:

Ввести user@domain.com.
Нажать кнопку «Проверить»
Ожидаемый результат: ошибка «Введен некорректный email»

Блок 3. Граничные и особые случаи

14. Пустое поле ввода

Шаги:

Оставить поле ввода пустым
Нажать кнопку «Проверить»
Ожидаемый результат: ошибка «Введен некорректный email»

15. Ввод только пробелов

Шаги:

Ввести несколько пробелов (например, )
Нажать кнопку «Проверить»
Ожидаемый результат: ошибка «Введен некорректный email»

16. Допустимый символ «_» в локальной части

Шаги:

Ввести user_name@example.com
Нажать кнопку «Проверить»
Ожидаемый результат: alert «Прекрасный email-адрес» (так как подчёркивание разрешено)

17. Допустимый символ «+» в локальной части

Шаги:

Ввести user+tag@example.com
Нажать кнопку «Проверить»
Ожидаемый результат: alert «Прекрасный email-адрес»

18. Слишком длинный email (граница длины – например, 256 символов)

Шаги:

Ввести email длиной 255 символов (валидный по структуре)
Нажать кнопку «Проверить»
Ожидаемый результат: alert или ошибка – зависит от реализации. По умолчанию предположим, что ограничений нет, поэтому alert.

(Если есть требование на максимальную длину, можно добавить отдельный кейс – здесь не указано, поэтому опускаем.)

Блок 4. Валидация только по клику

19. Отсутствие валидации при вводе (до нажатия кнопки)

Шаги:

Ввести невалидный email (например, test)
Не нажимать кнопку «Проверить»
Ожидаемый результат: ошибка не появляется, поле ввода не подсвечивается, никаких сообщений нет.
