# 1.3.4-Semin

Тест-кейсы для формы валидации email

Блок 1. Позитивные сценарии (валидные email-адреса)
1. Ввод валидного email с латинскими буквами

Шаги:

1.Ввести в поле ввода user@example.com
2. Нажать кнопку «Проверить»
3.Ожидаемый результат: появляется alert с текстом «Прекрасный email-адрес»

2. Ввод валидного email с цифрами в локальной части

Шаги:

1.Ввести user123@domain.ru
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: alert «Прекрасный email-адрес»

3. Ввод валидного email с допустимыми спецсимволами

Шаги:

1.Ввести user!name#mail@example.org
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: alert «Прекрасный email-адрес»

4. Ввод валидного email с точкой в локальной части

Шаги:

1.Ввести first.last@sub.domain.com
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: alert «Прекрасный email-адрес»

5. Ввод валидного email с дефисом в домене

Шаги:

1.Ввести user@my-domain.co.uk
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: alert «Прекрасный email-адрес»

Блок 2. Негативные сценарии (невалидные email-адреса)

6. Отсутствие символа «@»

Шаги:

1.Ввести userexample.com
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: под полем ввода (над кнопкой) появляется красная ошибка «Введен некорректный email»

7. Отсутствие домена после «@»

Шаги:

1.Ввести user@
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: ошибка «Введен некорректный email»

8. Отсутствие локальной части до «@»

Шаги:

1.Ввести @example.com
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: ошибка «Введен некорректный email»

9. Наличие недопустимых символов (кириллица)

Шаги:

1.Ввести пользователь@mail.ru
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: ошибка «Введен некорректный email»

10. Наличие пробелов внутри email

Шаги:

1.Ввести user name@domain.com
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: ошибка «Введен некорректный email»

11. Две точки подряд в домене

Шаги:

1.Ввести user@domain..com
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: ошибка «Введен некорректный email»

12. Отсутствие точки в домене

Шаги:

1.Ввести user@domaincom
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: ошибка «Введен некорректный email»

13. Email заканчивается точкой

Шаги:

1.Ввести user@domain.com.
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: ошибка «Введен некорректный email»

Блок 3. Граничные и особые случаи

14. Пустое поле ввода

Шаги:

1.Оставить поле ввода пустым
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: ошибка «Введен некорректный email»

15. Ввод только пробелов

Шаги:

1.Ввести несколько пробелов (например, )
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: ошибка «Введен некорректный email»

16. Допустимый символ «_» в локальной части

Шаги:

1.Ввести user_name@example.com
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: alert «Прекрасный email-адрес» (так как подчёркивание разрешено)

17. Допустимый символ «+» в локальной части

Шаги:

1.Ввести user+tag@example.com
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: alert «Прекрасный email-адрес»

18. Слишком длинный email (граница длины – например, 256 символов)

Шаги:

1.Ввести email длиной 255 символов (валидный по структуре)
2.Нажать кнопку «Проверить»
3.Ожидаемый результат: alert или ошибка – зависит от реализации. По умолчанию предположим, что ограничений нет, поэтому alert.

(Если есть требование на максимальную длину, можно добавить отдельный кейс – здесь не указано, поэтому опускаем.)

Блок 4. Валидация только по клику

19. Отсутствие валидации при вводе (до нажатия кнопки)

Шаги:

1.Ввести невалидный email (например, test)
2.Не нажимать кнопку «Проверить»
3.Ожидаемый результат: ошибка не появляется, поле ввода не подсвечивается, никаких сообщений нет.
